# Other lists of postmortems — 14 postmortems

External meta-resources (lists/analyses of postmortems beyond this corpus).

- **AWS Post-Event Summaries** (`aws-post-event-summaries`) — AWS's official archive of post-event summaries for major AWS service disruptions.  <https://aws.amazon.com/premiumsupport/technology/pes/>
- **Availability Digest website** (`availability-digest-website`) — Archived magazine-style analyses of high-availability failures and outages across industries.  <https://web.archive.org/web/20190914025003/http://www.availabilitydigest.com:80/articles.htm>
- **Postmortems community** (`postmortems-community`) — Community-curated postmortem link collection, including the imported archive of the defunct Google+ postmortems community.  <https://web.archive.org/web/20230329091806/https://postmortems.info/>
- **John Daily's list of postmortems (in json)** (`john-daily-s-list-of-postmortems-in-json`) — Machine-readable JSON dataset of service postmortem links on GitHub.  <https://github.com/macintux/Service-postmortems>
- **Jeff Hammerbacher's list of postmortems** (`jeff-hammerbacher-s-list-of-postmortems`) **[dead link]** — Quora-hosted curated list of notable postmortem write-ups.  <https://www.quora.com/Jeff-Hammerbacher/Post-mortems>
- **NASA lessons learned database** (`nasa-lessons-learned-database`) — NASA's searchable Lessons Learned Information System of engineering and mission failure lessons.  <https://llis.nasa.gov/>
- **Tim Freeman's list of postmortems** (`tim-freeman-s-list-of-postmortems`) — Pinboard tag feed continuously aggregating postmortem links from around the web.  <https://pinboard.in/u:peakscale/t:postmortem/>
- **Wikimedia's postmortems** (`wikimedia-s-postmortems`) — Wikimedia's public incident documentation: full reports for outages of Wikipedia-family infrastructure.  <https://wikitech.wikimedia.org/wiki/Incident_documentation>
- **Autopsy.io's list of Startup failures** (`autopsy-io-s-list-of-startup-failures`) — Archived catalog of startup failure stories — business postmortems rather than technical outages.  <https://web.archive.org/web/20190101075846/http://autopsy.io/>
- **SRE Weekly** (`sre-weekly`) — Weekly SRE newsletter whose recurring Outages section tracks fresh incidents and postmortems.  <https://sreweekly.com>
- **Lorin Hochstein's list of major incidents** (`lorin-hochstein-s-list-of-major-incidents`) — Curated GitHub list of major, high-profile software incidents with links to primary sources.  <https://github.com/lorin/major-incidents>
- **Awesome Tech Postmortems** (`awesome-tech-postmortems`) — Awesome-list style GitHub collection of tech postmortems organized by failure category.  <https://github.com/snakescott/awesome-tech-postmortems>
- **Nat Welch's parsed postmortems** (`nat-welch-s-parsed-postmortems`) — Project that parses this markdown list into a structured, queryable postmortem database.  <https://github.com/icco/postmortems>
- **Postmortem Templates** (`postmortem-templates`) — Collection of postmortem templates from various organizations for writing your own incident reports.  <https://github.com/dastergon/postmortem-templates>
